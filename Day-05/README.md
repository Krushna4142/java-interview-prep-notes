## ğŸ§© REST Design Principles

Professional REST APIs follow:

- Use nouns, not verbs in URLs
- Use plural resource names
- Use correct HTTP methods
- Maintain statelessness
- Return proper HTTP status codes

Example:

GET /users  
POST /users  
GET /users/101  
PUT /users/101  
DELETE /users/101

Bad Example:

GET /getAllUsers  
POST /createUser

REST prefers resource-based structure.
